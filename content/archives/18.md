下面简单介绍下PHP项目分布式部署中，SESSION的同步方案中的一种，使用Memcache来存储SESSION。至于Memcached的安装请看[《CentOS-6.4安装Memcached》](http://blog.phpha.com/archives/17/)。


<!--more-->


1、直接修改php.ini配置文件

```ini
# blog.phpha.com
session.save_handler = memcache
session.save_path = "tcp://127.0.0.100:11211"
# blog.phpha.com
```

2、利用目录下的 .htaccess 文件

```
php_value session.save_handler "memcache"
php_value session.save_path "tcp://127.0.0.1:11211"
```

说明：这个只是针对Apache的，目前使用Nginx的比较多，也不推荐此方式。

3、项目中修改配置

```php
// blog.phpha.com
ini_set("session.save_handler", "memcache");
ini_set("session.save_path", "tcp://127.0.0.100:11211");
// blog.phpha.com
```

上面只是一些简单的说明，其实说点题外话，一般来说，做分布式，那肯定是有服务器权限的，所以推荐第一种。
